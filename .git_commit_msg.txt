전체 변경 요약

문서/설정/코드 정합성 중심으로 정리했고, api_result는 B안(현재 미사용 기능으로 간주, 문서/설정/코드 잔재 제거) 을 선택했습니다.
설정 로딩은 *_env 간접참조 방식 없이, config.yaml 기본값 + 환경변수 override 방식으로 통일했습니다.
.vscode, __pycache__, 로그/임베딩/chroma 산출물 등 저장소에 추적되면 안 되는 항목을 제거/무시 처리했습니다.
PromptManager, config_loader를 단순화했고, 최소 자동 테스트(unittest)를 추가했습니다.
검증: python -m unittest discover -s tests -p "test_*.py" -v 통과(9개), python -m compileall Root_Stream Root_Ingest Planner tests 실행 완료.

삭제한 파일/폴더 목록과 삭제 이유
.vscode/launch.json, .vscode/settings.json
이유: 개인 IDE 설정 파일, 저장소 추적 불필요.
Api_test/api_test_client.py
이유: 현재 구조 기준 유지가치 낮고 역할 중복.
Root_Stream/services/api/ (모듈 파일 포함)
이유: api_result 제거 결정(B안)에 따라 미사용 경로 정리.
Root_Stream/**/__pycache__, Root_Ingest/**/__pycache__, *.pyc 다수
이유: 빌드 캐시/자동생성 산출물.
Root_Stream/data/logs/stream.log, Root_Ingest/data/logs/ingest.log, Root_Ingest/data/chroma/chroma.sqlite3, Root_Ingest/data/{chunks,embeddings,parsed}/*
이유: 실행 산출물/로컬 데이터, 버전관리 대상 아님.
